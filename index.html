01. Abstract

CivicEye is an AI-based system for automatic civic issue reporting. It allows citizens to capture images with location using a web interface. The system uses YOLOv8 to detect potholes, calculate damage severity, and assign priority levels. It also estimates repair cost and manpower, and automatically sends complaint details to government authorities. A hybrid location method (GPS, EXIF, OCR) ensures accurate location detection. This system reduces manual work and speeds up the reporting process to under 30 seconds.

02. Domain

Artificial Intelligence & Computer Vision

04. Existing System

Existing civic applications such as Namma Salai, FixMyStreet, and Civic Reporter allow users to report issues using images and GPS location. However, these systems rely on manual verification and lack AI-based analysis. There is no automatic severity calculation, prioritization, or cost estimation. As a result, complaint processing is slow and often takes several days or weeks.

06. Limitations of Existing System
  • No quantitative severity assessment – equal weight to all complaints.
  • Manual routing & delays – weeks of latency.
  • No cost/manpower estimation – separate site assessment.
  • Weak geo-tagging – user typed addresses often inaccurate.
  • Unprocessed visual evidence – images never analyzed.
07. Proposed System: Two‑Module Architecture

Citizen Module

Camera capture, Geolocation API, description + instant submit.

Government AI Module

YOLOv8 detection, severity scoring, cost/manpower, SMTP dispatch.
